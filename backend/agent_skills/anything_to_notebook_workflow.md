# 任意内容转笔记工作流
triggers: NotebookLM, anything to notebook, 任意内容, 转笔记, 生成播客, 生成PPT, 思维导图, 生成Quiz, 深度分析, 多源内容, 微信文章, YouTube, PDF转笔记, EPUB, 文档转报告

## 目标

当用户发来网页、YouTube、PDF、EPUB、Markdown、Word、PPT、Excel、图片、音频、播客、X/Twitter 链接或搜索关键词，并要求“生成播客 / PPT / 思维导图 / Quiz / 报告 / 深度分析 / 传到 NotebookLM / 整理成资料”时，按多源内容处理流水线执行。

本技能参考 `qiaomu-anything-to-notebooklm` 的产品思路，但在 BKLT 黑光里只接入合规的公开内容处理、文件转换、知识整理和生成规划；不做登录绕过、权限绕过或付费内容规避。

## 内容源识别

- YouTube：优先作为 URL 源交给 NotebookLM 或本地网页/字幕能力处理，不主动下载受限内容。
- 普通公开网页：使用 `local_scrape_url`、`browser_navigate` 或 `web_search` 获取正文。
- X/Twitter：先尝试公开页面或搜索；若登录墙不可读，让用户发截图或正文。
- PDF / Markdown / TXT / Office / EPUB：优先读取本地文件；必要时转成 Markdown/TXT 再进入知识库。
- 图片 / 扫描件：如果项目已有 OCR 能力则提取文字；否则作为待接入能力记录。
- 音频 / 播客：如果已有转写能力则转文本；否则登记为“转写工具需求”。
- 搜索关键词：先搜索并汇总公开来源，再生成报告或笔记。

## 意图映射

| 用户说法 | BKLT 内部意图 | 输出 |
| --- | --- | --- |
| 生成播客 / 做成音频 | audio_brief | 播客脚本或 TTS 任务 |
| 做成 PPT / 幻灯片 | slide_deck | 大纲、页结构或 PPT 生成任务 |
| 思维导图 / 脑图 | mind_map | Mermaid / JSON / Markdown 脑图结构 |
| Quiz / 出题 / 测验 | quiz | 题目、答案、解析 |
| 报告 / 总结 / 文档 | report | Markdown 报告 |
| 深度分析 / 递归提问 | deep_analysis | 多轮问题、答案、洞察和行动项 |
| 传到 NotebookLM | notebooklm_source | NotebookLM 集成任务或待办 |

## 执行步骤

1. 确认用户输入类型：URL、本地路径、文本、搜索词、混合多源。
2. 获取公开可访问内容，或读取用户本地文件。
3. 如果无法读取，要求用户补截图、正文或本地文件，不猜测内容。
4. 清洗内容：标题、来源、时间、正文、关键链接、表格、引用。
5. 根据意图生成目标结构：报告、PPT 大纲、播客脚本、脑图、Quiz、深度分析问题。
6. 可以调用 `notebook_ingest` 将整理后的文本写入 BKLT 知识库。
7. 若用户明确要 NotebookLM，而本地还没有 NotebookLM 工具，则创建集成待办：认证方式、上传 source、生成 artifact、下载结果。

## 深度分析模板

对长文、书、视频转写、播客转写，默认三轮：

1. 概览：主题、结构、核心观点、关键事实。
2. 深挖：论证逻辑、反方观点、风险点、缺失信息。
3. 反刍：对用户有什么用、能落地什么行动、该加入哪个技能/工作流。

## 风险边界

- 只处理公开内容、用户提供的内容、用户本地文件或用户有权访问的内容。
- 不帮助绕过登录、订阅、权限、访问控制或平台限制。
- 涉及版权内容时，只做摘要、结构化笔记和用户个人学习用途提醒。
- 涉及投资、空投、钱包、合约、账户授权时，只做风险识别和尽调清单，不引导连接钱包、授权或转账。

