# 社交内容转技能
triggers: x.com, twitter, 推文, X链接, 社交内容, 技能接入, 接进去技能, 看到这个, 学一下这个, 知识转技能, social source, tweet to skill

## 目标

当用户发来 X/Twitter、网页、帖子、长文、截图或一段外部内容，并说“接进去技能”“学一下”“做成能力”“加到黑光里”时，把内容转成 BKLT 黑光可复用的技能、工具需求或知识库资料，而不是只做普通总结。

## 执行流程

1. 先获取内容
   - 优先使用 local_scrape_url、browser_navigate 或 web_search 获取公开正文。
   - 如果页面因为登录、权限或访问限制不可读，明确告诉用户“当前没拿到原文”，让用户发截图或复制正文。
   - 不要根据 URL 猜内容；没拿到正文就只能先建待接入记录和通用模板。

2. 判断接入类型
   - 如果内容是操作方法、提示词、工作流、技巧：接成 backend/agent_skills/*.md 技能。
   - 如果内容是可自动执行的动作：先写工具需求文档，再评估是否新增后端工具。
   - 如果内容是资料、名单、案例、观点：优先用 notebook_ingest 写入知识库。
   - 如果内容涉及资金、账户、钱包、平台规则或敏感操作：只做风险识别、安全检查和合规提醒，不直接执行高风险动作。

3. 技能文件格式
   - 文件名使用小写英文和下划线，例如 social_source_to_skill.md。
   - 第一行是 # 技能标题。
   - 第二行是 triggers:，用中文和英文关键词混合，方便召回。
   - 正文必须包含：目标、适用场景、步骤、输入要求、输出格式、风险边界。

4. 输出给用户
   - 先说明是否成功读取原文。
   - 再说明接成了哪种类型：技能 / 工具需求 / 知识库 / 待接入。
   - 如果已改仓库，必须给出文件路径和提交号。
   - 如果还缺原文，直接让用户发截图或复制正文，不要让用户理解工程细节。

## 输入要求

用户可能只发一个链接，也可能发“看看这个接进去”。Agent 应主动处理，不要要求用户填写表单。

## 输出格式建议

```text
已处理：<链接或内容名>
接入类型：技能 / 工具需求 / 知识库 / 待接入
落地位置：<文件路径>
还缺：<如果没拿到正文，说明需要截图或原文>
下一步：<一个最短动作>
```

## 风险边界

- 不帮助获取非公开内容，不绕过登录或权限限制。
- 不把未经核实的投资、空投、NFT、加密项目宣传直接做成推荐策略。
- 涉及交易、私钥、钱包授权、空投领取、合约交互时，只能做风险检查和安全提醒，不能诱导用户连接钱包或转账。
- 可以做项目尽调流程、内容提炼模板、骗局识别清单、监控与提醒工作流。
