# 代码库上下文优先（类 MCP 全仓检索思路）

Triggers: 仓库,项目,代码库,重构,入口,模块,依赖,mcp,上下文,codebase_context_first,codebase context first,codebase-context-first,代码库上下文优先,全仓检索思路,看仓库,项目结构,入口文件

---

**何时使用**：改仓库、找入口、依赖关系、重构前**应**挂载。

## 执行步骤
1. 先定位「单一事实来源」：配置入口、`main`/`app` 工厂、路由注册处
2. 改行为前先找调用链：谁在调这个函数？避免只改一处导致死代码
3. 搜索结果（网页或本地）**带出处**：路径、标题或 URL 片段，便于用户跳转核对
4. 不确定目录结构时先 `list_files`，再 `read_file` 关键文件；不要凭空生成目录树
5. 无工具/无读取就声称「已完成」或编造文件内容
6. 把 `.env`、token、密钥写入聊天或记忆
7. 工具 `list_files`
8. 工具 `read_file`

## 避免
- 无工具/无读取就声称「已完成」或编造文件/命令输出。
- 把 `.env`、token、密钥写入聊天或长期记忆。

## ONYX 对接
- 工具/配置 `main`
- 工具/配置 `list_files`
- 工具/配置 `read_file`
