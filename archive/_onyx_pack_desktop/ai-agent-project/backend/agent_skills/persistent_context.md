# 持久上下文与记忆

Triggers: 记忆,回忆,上下文,偏好,习惯,复盘,playbook,长期,持久化,跨会话,benchmark,persistent_context,persistent context,persistent-context,持久上下文与记忆,你记得吗,根据历史,上次对话,我的习惯

---

**何时使用**：长期偏好、跨会话记忆、playbook、用户说「记住/以后默认」时**应**挂载。

## 执行步骤
1. 用户显式说「记住」「以后默认」时，用自然语言复述确认，再落盘到可执行结论
2. 回答前先假设：长期记忆与 playbook 可能不完整；缺证据就查工具，不要脑补用户历史
3. 同一事实只保留一条高置信表述，避免在答案里堆叠重复摘要
4. **去重与新鲜度**：新记忆与旧条冲突时，标注「取代/补充」关系；尽量带来源或日期，避免永久陈旧假设
5. **可检索**：写入时用最短可区分关键词，便于后续按任务类型命中（少而精、可评测召回）
6. 无工具/无读取就声称「已完成」或编造文件内容
7. 把 `.env`、token、密钥写入聊天或记忆
8. （示例）用户会用自然语言提到「持久上下文与记忆」相关任务

## 避免
- 无工具/无读取就声称「已完成」或编造文件/命令输出。
- 把 `.env`、token、密钥写入聊天或长期记忆。
