## 执行步骤
1. **多源交叉**：同一结论至少两类来源（例如网页 + 论文预印本，或官方文档 + 源码）再写进最终答案
2. **本地优先**：能读用户仓库/附件就不要只依赖模糊回忆；大文件先定位章节再摘录
3. **不确定性**：检索不到就写「未找到」+ 已尝试的关键词/路径，不编造条目
4. **隐私**：敏感材料默认最小暴露原则，日志式引用用摘要替代全文粘贴
5. 无工具/无读取就声称「已完成」或编造文件内容
6. 把 `.env`、token、密钥写入聊天或记忆
7. 将内网/隐私数据发到外网检索
8. （示例）用户会用自然语言提到「本地与可溯源深度检索」相关任务

## 避免
- 无工具/无读取就声称「已完成」或编造文件/命令输出。
- 把 `.env`、token、密钥写入聊天或长期记忆。
- 将内网/隐私/凭证发到外网检索。
